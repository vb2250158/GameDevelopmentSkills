# project-progress-pm

基于腾讯文档项目进度表做轻量 PM 巡检，识别风险、整理待办、生成催办话术，并在用户确认后写回进度结论。

## 需要准备

- 项目进度腾讯文档链接，例如 `https://docs.qq.com/sheet/<file_id>?tab=<sheet_id>`。
- 可用的 `read-tencent-docs-opendoc` skill 或腾讯文档 MCP。
- 若要写回表格，需要配置 `TENCENT_DOCS_TOKEN` 并获得用户明确授权。
- 若要发送 QQ/NapCat 消息，需要可用的聊天机器人接入和明确发送授权。
- 本地 PM 缓存目录，用于保存 message_id、负责人映射、等待链和下一步动作摘要。

## 环境变量

```powershell
$env:TENCENT_DOCS_TOKEN = "<token>"
```

如果接入 QQ/NapCat 或其他聊天平台，平台 token、host、群号等按具体机器人项目配置，不应提交到仓库。

## 典型巡检输入

```text
根据这个腾讯文档链接巡检项目进度，列出需要跟进的人、事项、风险和催办话术：
https://docs.qq.com/sheet/<file_id>?tab=<sheet_id>
```

## 输出内容

- 当前未闭环事项和状态分布。
- 卡住、待验收、信息不足或需要换人跟进的事项。
- 按负责人分组的短催办话术。
- 建议写回腾讯文档的 `修正方案批注`、`状态` 或数据质量备注。
- 当前等待谁、等待多久、下一步推进窗口。

## 使用边界

- 默认只读表格并生成报告。
- 默认只生成待审催办话术，不自动发消息。
- 默认只提出写表建议，不自动写回。
- 只有用户明确确认目标单元格和值后，才写回腾讯文档并读回校验。
